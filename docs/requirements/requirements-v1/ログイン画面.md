# 要件定義書: ログイン画面（P001）

## 目的

グループ会社の従業員がGoogle OAuthでログインし、システムにアクセスする。許可ドメインのアカウントのみログインできる。

## 機能

| # | 機能 | 説明 | v1からの変更点 |
|---|------|------|--------------|
| 1 | Googleログイン | Google OAuth 2.0 + PKCEフローでログインする | 新規 |
| 2 | ドメイン制限 | 許可ドメインのみログイン可能 | 新規 |
| 3 | 自動リダイレクト | ログイン済みならダッシュボードへ自動遷移 | 新規 |

## Input / Process / Output

| 機能 | Input | Process | Output |
|------|-------|---------|--------|
| Googleログイン | ユーザーのGoogleアカウント（ボタンクリック） | Supabase Auth経由でGoogle OAuth 2.0 + PKCEフロー実行。コールバックでセッションCookie設定、プロバイダトークンをDB保存 | ログイン成功→ダッシュボードへ遷移。失敗→エラーメッセージ表示 |
| ドメイン制限 | ログインしたGoogleアカウントのメールドメイン | ドメインが許可リスト（boostconsulting.co.jp, boostcapital.co.jp）に含まれるか検証 | 許可ドメイン→ログイン続行。非許可ドメイン→エラー「このドメインではログインできません」 |
| 自動リダイレクト | ページ読み込み時のセッション情報 | セッションCookieの有効性を確認 | 有効なセッションあり→ダッシュボードへリダイレクト。なし→ログイン画面表示 |

## UI状態遷移

| 要素 | 対応機能 | 状態 |
|------|---------|------|
| Googleでログインボタン | Googleログイン | デフォルト / ローディング（OAuth中） / 無効（エラー時） |
| エラーメッセージ | ドメイン制限 | 非表示 / 表示（ドメインエラー・認証エラー） |
| ページ全体 | 自動リダイレクト | ログイン画面表示 / リダイレクト中（スピナー表示） |
