# Input / Process / Output 一覧

> 全機能のデータの流れと処理の内容が網羅される

## IPO 一覧

| # | 機能 | Page | Input | Process | Output |
|---|------|------|-------|---------|--------|
| 1 | Googleログイン | P001 | Googleアカウント（ボタンクリック） | Supabase Auth経由でGoogle OAuth 2.0 + PKCEフロー実行。コールバックでセッションCookie設定、プロバイダトークンをDB保存 | ログイン成功→ダッシュボードへ遷移。失敗→エラーメッセージ表示 |

## データフロー確認

### 主要なデータの流れ

```
#1 Googleログイン → [セッション・トークン] → #3 自動リダイレクト, #4 リンク一覧表示, #9 Calendar連携ステータス
#11-19 リンク作成設定 → [スケジュールリンク] → #20 リンク生成・コピー → #4 リンク一覧表示
#20 リンク生成 → [公開URL] → #6 URLコピー → [メール/SNSで共有] → #22 空き時間スロット表示
#12b カレンダー連携招待 → [招待メール] → 外部ユーザーがGoogleログイン → [GUESTユーザー・トークン保存] → #15 空き時間自動計算に含まれる
#16 会議形式選択 + #17 バッファー時間設定 → [形式別バッファー設定] → #27 会議モード選択 → #22 空き時間スロット表示（再計算）
#22-29 予約情報 → #30 予約確定 → [予約データ] → #31 予約完了表示, #33 リマインダーメール送信
#15 空き時間自動計算 ← [Googleカレンダー予定] → #22 空き時間スロット表示
```

### データフローの整合性チェック

| チェック項目 | 結果 |
|------------|------|
| すべてのOutputに対応するInputがあるか | ✅ #20のOutput（URL）→ #6のInput（コピー対象）→ #22のInput（スラッグ） |
| すべてのInputに提供元があるか | ✅ #22のInput（会議形式）← #27のOutput（選択結果） |
| データの型・形式に矛盾がないか | ✅ スラッグ文字列がURL生成からゲスト画面まで一貫 |
| バッファー設定が正しく伝播するか | ✅ #17（設定保存）→ リンク設定JSON → #27（形式切替時に参照）→ #22（スロット再計算） |
| リマインダーの入力データが存在するか | ✅ #30（予約確定）→ 予約データDB → #33（前日バッチで検索） |
