# 要件定義書 v2: 調整回答画面（P004）

> DB設計を踏まえた要件定義書の更新

## 目的

共有URLを受け取ったゲストが、空き時間スロットを確認して予約する。認証不要で利用可能。Google/Microsoftログイン連携（任意）で自分の予定をオーバーレイ表示できる。

## 機能

| # | 機能 | 説明 | DB関連テーブル |
|---|------|------|--------------|
| 1 | 空き時間スロット表示 | 主催者の空き時間をカレンダービューで表示 | schedule_links (READ: slug), users (READ: トークン) |
| 2 | 日付選択 | カレンダーから予約したい日付を選択 | なし（フロント計算） |
| 3 | 時間スロット選択 | 利用可能な時間枠を選択 | なし（フロント状態） |
| 4 | 会議時間選択 | 許可された会議時間から選択 | schedule_links.settings (allowedDurations) |
| 5 | カレンダー連携（任意） | Google/MSログインで自分の予定をオーバーレイ表示 | なし（ゲスト側OAuth、DBに保存しない） |
| 6 | 会議モード選択 | オンライン/対面を選択。バッファーに応じてスロット再計算 | schedule_links.settings (meetingOptions) |
| 7 | 会議ツール指定（任意） | Zoom/Teams等のカスタムURLを入力 | なし（フロント入力、予約時に保存） |
| 8 | 予約情報入力 | 名前・メール・メモを入力 | なし（フロント入力、予約時に保存） |
| 9 | 予約確定 | 予約を確定し全員にカレンダー招待を送信 | bookings (CREATE), booking_participants (CREATE) |
| 10 | 予約完了表示 | 予約完了の確認情報を表示 | bookings (READ) |
| 11 | エラー表示 | 無効なリンクへのアクセス時にエラー表示 | schedule_links (READ: is_active) |

## Input / Process / Output

| 機能 | Input | Process | Output | DB関連テーブル |
|------|-------|---------|--------|--------------|
| 空き時間スロット表示 | URLスラッグ、選択中の会議形式 | APIでリンク情報取得。バッファー時間を加味して空き時間計算 | カレンダービューに空き日付をハイライト。ヘッダーにリンク情報表示 | schedule_links (READ WHERE slug = param AND is_active = true), users (READ: access_token for 参加者カレンダー取得) |
| 日付選択 | カレンダー上の日付クリック | 選択日の利用可能スロットをフィルタ | その日の時間枠リストを表示 | なし |
| 時間スロット選択 | 時間枠リストからクリック | 選択スロットを予約候補として保持 | ハイライト。予約フォーム表示 | なし |
| 会議時間選択 | 会議時間ボタン選択 | 主催者許可の時間のみ選択可。スロット再計算 | スロット一覧が更新 | schedule_links.settings.allowedDurations |
| カレンダー連携（任意） | Google/MSログインボタンクリック | OAuthフロー。ゲストのカレンダー予定を取得 | オーバーレイ表示（半透明） | なし（一時的なOAuth、DBに保存しない） |
| 会議モード選択 | 形式から選択 | バッファー時間が変わるため空きスロットをAPIで再取得 | スロット一覧が動的に更新。形式に応じたUI表示 | schedule_links.settings.meetingOptions (READ: buffer*) |
| 会議ツール指定（任意） | カスタム会議URL入力 | URL形式バリデーション | 予約情報に反映。未入力時はGoogle Meet自動生成 | なし（予約確定時にbookings.meeting_urlに保存） |
| 予約情報入力 | 名前（必須）、メール（必須）、メモ（任意） | バリデーション | フォームに値が反映 | なし（予約確定時に保存） |
| 予約確定 | 予約確定ボタンクリック | ダブルブッキングチェック。DB保存。Googleカレンダーにイベント作成。Google Meet URL生成。会議室予約。確認メール送信 | 成功→完了画面。ダブルブッキング→エラー | bookings (INSERT), booking_participants (INSERT for each 参加者), schedule_links (READ: settings.participants), users (READ: 参加者トークン) |
| 予約完了表示 | 予約確定成功 | 予約情報を整形 | 予約日時、会議情報、参加者一覧を表示 | bookings (READ) |
| エラー表示 | URLスラッグ | リンク取得し有効性判定 | エラー画面表示。予約UI非表示 | schedule_links (READ WHERE slug = param: 存在しない or is_active = false) |

## UI状態遷移

| 要素 | 対応機能 | 状態 | DB関連テーブル |
|------|---------|------|--------------|
| カレンダービュー | スロット表示、日付選択 | ローディング / 表示中 / エラー | schedule_links, users |
| 時間枠リスト | スロット選択 | 非表示 / 表示中 / 空 | なし |
| 会議時間ボタン | 会議時間選択 | 未選択 / 選択中 | schedule_links.settings |
| Google/MSログインボタン | カレンダー連携 | デフォルト / 連携済み | なし |
| オーバーレイ予定 | カレンダー連携 | 非表示 / 表示中 | なし |
| 会議モード選択 | 会議モード選択 | オンライン / 対面（社内） / 対面（訪問） / 非表示。切替時にスロット再読み込み | schedule_links.settings.meetingOptions |
| カスタム会議URL入力 | 会議ツール指定 | 非表示 / 表示中 / エラー | なし |
| 予約フォーム | 予約情報入力 | 非表示 / 表示中 / バリデーションエラー | なし |
| 予約確定ボタン | 予約確定 | 無効 / デフォルト / ローディング | bookings |
| 予約完了画面 | 予約完了表示 | 非表示 / 表示中 | bookings |
