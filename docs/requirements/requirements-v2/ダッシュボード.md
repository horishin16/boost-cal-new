# 要件定義書 v2: ダッシュボード（P002）

> DB設計を踏まえた要件定義書の更新

## 目的

ログイン後のホーム画面。作成済みのスケジュールリンク一覧を表示し、リンクの作成・編集・削除・URLコピーを行う。

## 機能

| # | 機能 | 説明 | DB関連テーブル |
|---|------|------|--------------|
| 1 | リンク一覧表示 | 自分が作成したスケジュールリンクの一覧を表示 | schedule_links (READ) |
| 2 | リンク新規作成 | 予定調整作成画面へ遷移 | なし（ルーティングのみ） |
| 3 | リンクURLコピー | 共有URLをクリップボードにコピー | schedule_links (READ: slug) |
| 4 | リンク編集 | 既存リンクの編集画面へ遷移 | なし（ルーティングのみ） |
| 5 | リンク削除 | スケジュールリンクを削除（確認あり） | schedule_links (DELETE) |
| 6 | Calendar連携ステータス | Google Calendarとの連携状態を表示 | users (READ: access_token) |
| 7 | ログアウト | セッション終了、ログイン画面へ戻る | なし（セッション操作のみ） |

## Input / Process / Output

| 機能 | Input | Process | Output | DB関連テーブル |
|------|-------|---------|--------|--------------|
| リンク一覧表示 | ログインユーザーのセッション | APIで自分のスケジュールリンク一覧を取得 | リンクカード一覧（名前、URL、参加者数、作成日） | schedule_links WHERE owner_id = current_user |
| リンク新規作成 | 新規作成ボタンクリック | P003へルーティング | P003が新規作成モードで表示 | なし |
| リンクURLコピー | URLコピーボタンクリック | 公開URLをクリップボードにコピー | トースト「URLをコピーしました」表示 | schedule_links.slug からURL構築 |
| リンク編集 | 編集ボタンクリック（対象リンクID） | P003へリンクIDと共にルーティング | P003が編集モードで表示 | なし |
| リンク削除 | 削除ボタンクリック → 確認ダイアログで「削除」 | APIでスケジュールリンクを削除 | 一覧から該当カードが消える。トースト表示 | schedule_links DELETE WHERE id = target AND owner_id = current_user |
| Calendar連携ステータス | ページ読み込み時 | Googleアクセストークンの有効性を確認 | 連携済み→正常表示。未連携→警告バナー表示 | users.access_token の有効性確認 |
| ログアウト | ログアウトボタンクリック | セッションCookie削除、Supabaseセッション終了 | P001へリダイレクト | なし |

## UI状態遷移

| 要素 | 対応機能 | 状態 | DB関連テーブル |
|------|---------|------|--------------|
| リンクカード一覧 | リンク一覧表示 | ローディング / 一覧表示 / 空状態 | schedule_links |
| 新規作成ボタン | リンク新規作成 | デフォルト / 無効（Calendar未連携時） | users (access_token有無) |
| URLコピーボタン | URLコピー | デフォルト / コピー完了 | schedule_links (slug) |
| 編集ボタン | リンク編集 | デフォルト | なし |
| 削除ボタン | リンク削除 | デフォルト / 確認ダイアログ表示中 | なし |
| 削除確認ダイアログ | リンク削除 | 非表示 / 表示 | なし |
| Calendar連携バナー | Calendar連携ステータス | 非表示（連携済み） / 警告表示（未連携） | users (access_token) |
