# 要件定義書 v2: ログイン画面（P001）

> DB設計を踏まえた要件定義書の更新

## 目的

グループ会社の従業員がGoogle OAuthでログインし、システムにアクセスする。許可ドメインのアカウントのみログインできる。

## 機能

| # | 機能 | 説明 | DB関連テーブル |
|---|------|------|--------------|
| 1 | Googleログイン | Google OAuth 2.0 + PKCEフローでログインする | users (CREATE/UPDATE) |
| 2 | ドメイン制限 | 許可ドメインのみログイン可能 | users (READ: domain) |
| 3 | 自動リダイレクト | ログイン済みならダッシュボードへ自動遷移 | なし（セッションCookieのみ） |

## Input / Process / Output

| 機能 | Input | Process | Output | DB関連テーブル |
|------|-------|---------|--------|--------------|
| Googleログイン | Googleアカウント（ボタンクリック） | Supabase Auth経由でGoogle OAuth 2.0 + PKCEフロー実行。コールバックでセッションCookie設定、プロバイダトークンをDB保存 | ログイン成功→ダッシュボードへ遷移。失敗→エラーメッセージ表示 | users: email, name, domain, google_id, access_token, refresh_token を INSERT or UPDATE |
| ドメイン制限 | ログインしたGoogleアカウントのメールドメイン | ドメインが許可リスト（boostconsulting.co.jp, boostcapital.co.jp）に含まれるか検証 | 許可→ログイン続行。非許可→エラー表示 | users: domain を検証（GUESTロールはドメインチェックをバイパス） |
| 自動リダイレクト | ページ読み込み時のセッション情報 | セッションCookieの有効性を確認 | 有効→ダッシュボードへリダイレクト。なし→ログイン画面表示 | なし |

## UI状態遷移

| 要素 | 対応機能 | 状態 | DB関連テーブル |
|------|---------|------|--------------|
| Googleでログインボタン | Googleログイン | デフォルト / ローディング（OAuth中） / 無効（エラー時） | なし |
| エラーメッセージ | ドメイン制限 | 非表示 / 表示（ドメインエラー・認証エラー） | users (domain検証結果) |
| ページ全体 | 自動リダイレクト | ログイン画面表示 / リダイレクト中（スピナー表示） | なし |
