# Integration Checker

フルスタック実装の統合テスト・動作確認を行う。

## Purpose

Phase 5：E2E テスト実行、ブラウザでの動作確認、パフォーマンス・アクセシビリティ確認。

## Inputs

- `__tests__/` - 全テストファイル
- `app/` - 全アプリケーションコード

## Outputs

- 統合テスト実行レポート
- 動作確認レポート
- パフォーマンス測定結果

## Procedure

### Step 1: ユニット・統合テスト実行

```bash
npm test
```

- すべてのテストが通ることを確認
- カバレッジ確認（必要に応じて）

### Step 2: ビルド・型チェック

```bash
npm run type-check
npm run lint
npm run build
```

- TypeScript エラーなし
- Linter エラーなし
- ビルド成功

### Step 3: 開発サーバー起動・ブラウザ動作確認

```bash
npm run dev
```

- 実装機能が期待通りに動作しているか
- UI が正しく表示されているか
- エラーメッセージが適切に表示されるか
- フォーム入力・検証が動作するか
- API 呼び出しが成功しているか

### Step 4: パフォーマンス確認

- ページロード時間
- 初期描画性能（Core Web Vitals）
- API レスポンス時間

### Step 5: アクセシビリティ確認

- キーボード操作（Tab キー）が機能するか
- スクリーンリーダー対応を確認
- カラーコントラスト確認

### Step 6: 完了確認

ユーザーに結果を報告：
- テスト全体の合否
- 発見された問題（あれば）
- パフォーマンス指標
- 次のアクションの提案

## 注意事項

- テスト失敗時は実装者に報告して修正を依頼
- 動作確認時は複数ブラウザ・デバイスでの確認を推奨
- パフォーマンス課題があれば、具体的な改善提案を行う
- ユーザーの確認を受けてスライス実装完了とする
